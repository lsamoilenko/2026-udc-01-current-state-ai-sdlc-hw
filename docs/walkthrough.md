**Тема:** Сучасний стан розробки з AI (Modern AI-assisted Development)
**Формат:** greenfield — ви створюєте власний невеликий проєкт
**Час:** ~1-2 години
**Здача:** Pull Request у starter-repo (CodeRabbit зробить авто-рев'ю)

## Task 1: Init проєкту + конфіг IDE _(~25 хв)_

**Мета:** запустити новий проєкт і налаштувати під нього Agentic IDE.

1. Ініціалізуйте невеликий проєкт у теці `app/`. Найпростіше — Next.js:
   ```bash
   npx create-next-app@latest app
   ```
   (можна будь-який інший стек — Vite/React, Express, тощо).
2. Додайте `app/AGENTS.md` із:
   - стеком і версіями;
   - командами (`dev` / `build` / `test`);
   - 3-5 конвенціями (нейминг, структура, стиль);
   - 2-3 guardrails (що НЕ чіпати, чого не робити).
3. Додайте `.cursorignore` (або `.claude/settings.json` з `permissions.deny`
   для Claude Code) — мінімум **3 патерни**, що прибирають з контексту
   `node_modules`, білди, lock-файли, `.env`.
4. Доставте мінімум один готовий **Agent Skill** під ваш стек. Для Next.js/React:
   ```bash
   npx skills add github.com/vercel-labs/agent-skills --skill vercel-react-best-practices
   ```
   (каталог скілів — [skills.sh](https://www.skills.sh/); інші стеки — оберіть доречний скіл).
   Скіл потрапить у `.agents/skills/` (або в теку вашого інструмента) — закомітьте його.
5. Перший запит до агента: «прочитай AGENTS.md і опиши проєкт». Переконайтеся,
   що контекст «зчитався».

> Skills — це окрема велика тема (Воркшоп 5). Тут достатньо встановити готовий
> і переконатися, що агент його бачить.

**Перевірка:** є `app/` (стартує `npm run dev`), `app/AGENTS.md`, `.cursorignore`,
встановлений скіл.

---

## Task 2: Фіча через режими _(~30 хв)_

**Мета:** реалізувати маленьку фічу, керуючи агентом через режими.

1. Оберіть **одну невелику фічу** (напр. список нотаток, лічильник, проста
   форма з валідацією).
2. Сформулюйте **acceptance criteria** (3-5 пунктів) **до** написання коду.
3. Пройдіть задачу у двох режимах:
   - **Plan** — попросіть план перед кодом, відрев'юйте, скоригуйте;
   - **Agent** — виконайте за погодженим планом.
4. Зробіть рев'ю дифу, запустіть, перевірте в браузері.
5. Опишіть процес у `docs/workflow.md`:
   - acceptance criteria фічі;
   - що робили в кожному режимі;
   - що скоригували після рев'ю плану.

**Перевірка:** фіча працює; `docs/workflow.md` конкретно описує plan → agent + рев'ю.

---

## Task 3: Аналіз токенів і вартості _(~25 хв)_

**Мета:** ПОМІРЯТИ, а не вгадувати вартість.

1. Зафіксуйте споживання токенів за фічу (usage-екран у вашому IDE) та/або
   запустіть `npx repomix`, щоб побачити розмір контексту в токенах.
2. Прикиньте вартість: `токени × ціна обраної моделі`.
3. Знайдіть **2-3 оптимізації** (вузький контекст, дешевша модель для рутини,
   коротші ітерації, чіткіші промпти).
4. (Опц.) повторіть фічу з оптимізацією і порівняйте.
5. Запишіть у `docs/cost-analysis.md`: числа + висновки.

**Перевірка:** `docs/cost-analysis.md` містить реальні числа й ≥2 висновки.

---

## Task 4 (bonus): A/B промптів _(~15 хв)_

**Мета:** відчути різницю базового і структурованого промпту.

1. Візьміть одну задачу і напишіть **два промпти**:
   - базовий («зроби X»);
   - структурований (роль + контекст + обмеження + acceptance criteria + формат).
2. Виконайте обидва, зафіксуйте різницю (якість, ітерації, токени).
3. Запишіть у `docs/ab-experiment.md`: обидва промпти + короткий висновок.

> Бонус — не обов'язковий, але це гарний розгін перед Воркшопом 2
> (промпт-інженерія).
